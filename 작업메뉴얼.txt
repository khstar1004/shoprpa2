

# RPA 관리담당자 업무 종합 매뉴얼

## 1. 업무흐름도 개요

### 1) 가격조사 대상 사이트
1. **고려기프트**  
2. **네이버**  
3. **네이버에 등록된 판촉물 업체**

### 2) 승인관리를 위한 가격조사 대상 사이트
1. **고려기프트**  
2. **네이버에 등록된 판촉물 업체**

### 3) 가격조사 프로세스 (승인관리 / 가격관리)

| 가격조사 단계 | **승인관리**             | **가격관리**            |
|----------------|---------------------------|--------------------------|
| 가격조사 1단계  | 구분 : **● 승인관리** / ○ 가격관리 | 구분 : ○ 승인관리 / **● 가격관리** |
| 가격조사 2단계  | RPA 실행 (가격조사)       | RPA 실행 (가격조사)       |
| 가격조사 3단계  | 가격조사 후 가격불량상품만 엑셀에 저장 | 동일 (가격불량상품만 저장)    |
| 가격조사 4단계  | 3단계 엑셀파일을 [상품가격관리] 게시판에 등록  | 동일 (3단계 파일 등록)     |
| 가격조사 5단계  | 가격불량상품 승인보류 (사유 : 가격 불량)  | -                      |

#### (A) 승인관리 상세 프로세스
1. **구분값을 [승인관리]**로 설정 후 RPA 실행  
2. 가격조사 결과파일을 홈페이지에 저장 (구분값 **A**)  
3. 2번 결과 중 **가격불량상품만** 추가로 엑셀에 분리 저장  
4. 3번에서 생성된 엑셀파일을 홈페이지의 [상품가격관리] 게시판에 등록  
   - 이때 구분값을 **[승인관리]**로 지정  
5. 등록된 가격불량상품을 홈페이지에서 **승인보류**

#### (P) 가격관리 상세 프로세스
1. **구분값을 [가격관리]**로 설정 후 RPA 실행  
2. 가격조사 결과파일을 홈페이지에 저장 (구분값 **P**)  
3. 2번 결과 중 **가격불량상품만** 추가로 엑셀에 분리 저장  
4. 3번에서 생성된 엑셀파일을 홈페이지의 [상품가격관리] 게시판에 등록  
   - 이때 구분값을 **[가격관리]**로 지정

## 2. 오후 업무 (준비 단계)

오후에는 다음날 오전에 돌릴 **RPA INPUT 파일**을 준비하고, RPA 실행 상태를 점검하는 업무를 주로 진행합니다.

### 2-1) RPA INPUT 파일 준비

1. **사이트 접속**  
   - `i.jclgift.com/admin`에 접속  
   - **‘상품관리’ → ‘전체상품목록’** 클릭
2. **‘품절상품여부’**를 **(상품준비중)** 으로 지정하고 검색
3. **‘가격조사엑셀’** 버튼 클릭 → 가격조사엑셀파일 다운로드  
4. **검색결과**와 **메신저 내역**(혹은 해당 상품 리스트) 대조하여 누락, 중복 여부를 확인

### 2-2) 가격조사엑셀파일 수정 및 분할

1. **엑셀파일 가공**  
   - 내려받은 가격조사 엑셀파일에 **‘구분’** 열을 추가(또는 기존 열 사용)하고, 모두 **`A`** 값을 입력  
   - (승인관리 업무이므로 ‘A’ 사용. 가격관리일 경우 ‘P’ 사용)
2. **상품명 수정**  
   1) 상품명에 **‘1-’** 이 포함된 경우 → ‘1-’ 및 앞의 숫자/하이픈 제거  
   2) 상품명에 **‘/’** 이나 **‘(번호)’** 등이 들어있는 경우 → 해당 특수문자, 괄호 등 제거
3. **파일 분할** (한 번에 300개 이상 처리 어려움)  
   - 예: 1019개 → 300개*3 + 119개*1 = 총 4개 파일  
   - 파일 이름 예시: `승인관리1(300)-날짜`, `승인관리2(300)-날짜` 등
   - 예: 562개 상품 → 300개, 262개로 분할

### 2-3) 완성된 파일을 각 RPA PC(PC1, PC2)의 INPUT 폴더에 배분

1. **RPA 중지**  
   - 두 PC(RPA PC1, PC2)에서 RPA가 실행 중이라면 중지
2. **config 파일 초기화**  
   - 각 PC의 `config` 파일에 입력된 숫자(기존 실행 중단 시점)를 모두 지워 **재실행 시 전체 행을 다시 인식**하도록 만듦
3. **INPUT/QUEUE 폴더 파일 정리**  
   - PC1과 PC2의 `INPUT` 폴더 및 `QUEUE` 폴더 안 파일을 모두 바탕화면으로 이동(백업)  
   - 편의상 PC1에서 다운받은 후, PC2 폴더로 복사/이동
4. **새 파일 배분**  
   - 2-2 단계에서 분할 완료된 파일을 **각 PC의 INPUT 폴더**에 하나씩 할당  
   - 나머지 파일은 각 PC의 `QUEUE` 폴더에 넣어준다 (PC1 성능이 더 좋아 조금 더 많은 수 할당 가능)
5. **RPA 실행**  
   - a.works 앱 실행 후, 왼쪽 상단의 줄3개 버튼으로 초기화 목록 확인  
   - `Local` 항목의 ▶ 버튼 클릭 → RPA 실행

### 2-4) 사장님께 보고

- 보고 문구 예시:
  ```
  승인관리 보고서
  1) 상태변경 상품수 : 577개
  2) 중복방지 상품수 : 15개
  3) 승인관리 상품수 : 562개
  4) RPA1 300개, RPA2 262개 입니다!
  ```

## 3. 오전 업무 (실행 단계)

아침에 출근 후, **전날 오후에 준비한 RPA INPUT 파일**이 정상적으로 돌고 있는지 확인하고, 승인관리/가격관리 작업을 마무리합니다.

### 3-1) 상품 승인 업무 시작 (상품준비중 상태 해제)

1. `i.jclgift.com/admin` 접속
2. **‘상품관리’ → ‘전체상품목록’** 클릭
3. 목록 수 100개로 지정, **품절상품여부 ‘상품준비중’** 선택
4. 전체 목록 선택 후 **‘준비상품해제’** 버튼 클릭  
   - ‘상품준비중’ 상태 일괄 해제

### 3-2) RPA Upload 파일 병합

1. **메일함 접속**  
   - 전날 RPA 실행 후 각 PC에서 전송된 결과 파일(예: PC1 400개, PC2 220개 등) 확인  
   - 여러 파일을 **하나의 Excel 파일**로 취합(병합)
2. **필터 조건**  
   - 특정 가맹점(공급사) 정보 제외  
   - 담당자 연차/휴가 등으로 인한 데이터 확인 후 필요 시 제외
3. **최종 파일 저장**  
   - 예: `승인관리(전체수-보류수)-YYYYMMDD_uplead_YYYYMMDD.xlsx`

### 3-3) 병합된 파일 해오름(Admin) 업로드

1. `i.jclgift.com/admin` → **‘상품관리 → 상품가격관리’** 이동
2. **‘가격조사파일 등록하기’** 클릭  
   - 병합한 RPA upload 파일 선택 → ‘등록’
   - 참고파일, 키워드/구분 일반적으로 생략
3. **적용 필터 설정**  
   - 관련검색어: ‘승인관리-거부’ → 적용  
   - 승인보류일: 오늘 날짜 → 적용  
   - 상태: ‘완료’ → 적용

### 3-4) 일정관리(메모) 등록

1. **‘일정관리’** 클릭 후,  
2. **‘상품승인관리(상품승인관리)’**로 관리대상 지정, 검색
3. 검색 결과의 **‘메모’** 클릭 → 메모등록 팝업
4. **메모등록**  
   - 제품수, 불량수  
   - 최근관리일: 오늘 날짜  
   - 최근구분: 승인관리  
   - 다음관리일: 다음 업무일  
   - 첨부파일: 3-2에서 병합한 파일  
   - 메모: ‘상품승인관리’ 등으로 기입  
   - **‘등록’** 버튼 클릭

### 3-5) 사장님께 보고

- 보고 문구 예시:
  ```
  안녕하세요! 승인관리 620개-318개 나와서 상품가격관리, 일정관리, 업로드 완료 했습니다!
  오늘 휴가자 (황성하주임) 2 보류건 9/10에 업로드 하겠습니다.
  어제 휴가자 (김지나대리) 2 건 업로드 했습니다.
  ```

# 요약: 전일(오후) 준비 → 익일(오전) 실행

1. **오후(준비)**  
   - 가격조사엑셀 다운로드 → ‘구분’ A/P 입력 → 상품명 정리 → 파일 분할(300건 단위)  
   - RPA PC1, PC2에 파일 배분, RPA 중지/재설정/재시작 → 보고
2. **오전(실행)**  
   - RPA 결과물(메일) 취합 → 필터 정리 → 해오름(Admin) 업로드  
   - 일정관리(메모등록) → 승인보류(필요 시) → 보고
